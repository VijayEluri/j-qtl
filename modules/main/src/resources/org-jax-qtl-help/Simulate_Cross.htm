Simulate Cross

The "Simulate Cross" dialog can be used to generate fake cross data for analysis. The dialog is essentially providing a graphical interface to the sim.map and sim.cross R/qtl functions. For detailed information on the parameters being set and the effects that they have see the R/qtl documentation.

 

First Panel (Settings for sim.map):

Most of the functionality on the first panel is used to edit the parameters being sent to sim.map.

 

Second Panel (Settings for sim.cross):

The second panel is used to set parameters for sim.cross.

 

Third Panel (Defining QTLs for sim.cross):

The third and final panel is used to define the QTL's that will be simulated as a part of the sim.cross function and naming your cross. You'll notice that the QTL effects that are available on the third panel depend upon the Cross Type that is selected on the second panel:

Cross Type: Available Effects:
Intercross Additive Effect & Dominance Deviation
Backcross Homozygote Effect - Heterozygote Effect
Four-Way Cross AC Effect, BC Effect  & AD Effect
